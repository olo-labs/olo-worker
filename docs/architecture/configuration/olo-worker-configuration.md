# olo-worker-configuration (configuration module)

This module **loads configuration during worker bootstrap**, maintains an **immutable in-memory snapshot**, and provides **fast read access during workflow execution**. Defaults are defined in `olo-defaults.properties`; environment variables (`OLO_*`) override these defaults.

---

## Configuration flow overview

```
    PostgreSQL
 (admin config DB)
        │
        ▼
 Admin Service
ConfigSnapshotBuilder
        │
        ▼
 Redis Snapshot
olo:configuration:* keys
        │
        ▼
 Worker Bootstrap
defaults → env → snapshot
        │
        ▼
 In-memory Snapshot
ConfigurationProvider
```

This gives readers instant context.

## Design principles

- **Workers never read configuration from the database**
- **Configuration is distributed as Redis snapshots**
- **Workers keep an immutable in-memory snapshot**
- **Snapshot refresh is incremental and safe**
- **Admin service is the single configuration writer**

---

## Why Redis snapshots?

This architecture ensures:

- **Consistent configuration** across all workers
- **Zero database load** from workers
- **Fast configuration reads** during workflow execution
- **Safe incremental configuration updates**

## Quick reference

### Workers

**Bootstrap.run()** loads configuration:

**defaults → env → Redis snapshot**

(If Redis is not configured, e.g. local development, workers use **defaults + environment variables** only.)

At runtime workers read **only from the in-memory snapshot** (no Redis/DB access):

- **ConfigurationProvider.require()**
- **TenantRegionResolver.getRegion(tenantId)**

### Admin

- **DB** → **ConfigSnapshotBuilder.buildAndStore(region)** → **Redis** (sectioned keys).
- Write order: data first, **meta last**. See [04_refresh](04_refresh.md) and [05_admin_service](05_admin_service.md).

### Redis snapshot keys

Redis snapshot keys store the **sectioned configuration snapshot** generated by the admin service. Formal schema (canonical key patterns): [03_snapshot_model](03_snapshot_model.md#formal-redis-schema).

```
olo:config:meta
olo:config:core
olo:config:pipelines:<region>
olo:config:connections:<region>
```
